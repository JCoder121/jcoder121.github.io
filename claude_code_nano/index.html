Claude Code Nano

Build your own Claude Code. A CS61A-style guided project: you implement a working ~350-line agent in TypeScript — the same core loop the real Claude Code runs — problem by problem, against a provided read-only test harness. Modeled on Hog.

There is no planner, no graph, no framework hiding the trick. An agent is a model that can ask for things, plus a harness that runs them and reports back, looped until the model stops asking. Six lectures, one file (index.ts), five sections, ten points. By the end you run your own agent against a real API and watch it find and fix a planted bug in a JavaScript file.

Who this is for

A programmer who thinks in data structures & algorithms first (arrays, queues, invariants, state machines) and games & simulations second (turn loops, game state, player moves). Fluent in programming but newer to TypeScript / Node specifically: async/await, promises, and Node APIs (fs, child_process, readline) are explained as they appear. Every abstract agent concept is anchored to a DSA idea — the conversation is an append-only array; the agent loop is a while-loop over a work queue; the edit tool’s uniqueness rule is an invariant check.

What you’ll be able to do

  • Explain and implement the agent loop from memory: call model → execute tool_use requests → append tool_results → repeat until no tool calls.
  • Implement the 5 core tools (read / list / write / edit / bash) and explain why edits use unique-string replacement.
  • Explain how permissions, denials, and tool errors flow back to the model as ordinary messages — and watch a model self-correct.
  • Run your own agent end-to-end: it finds and fixes a planted bug in fixtures/convert.js.
  • Read real Claude Code / agent-harness discussions and know exactly what the harness is doing.

Getting the starter files

Download starter.zip and unpack it. It contains the skeleton index.ts (fully-typed signatures with YOUR CODE HERE stubs), the read-only test harness (tests/, fixtures/), the ok runner, and the Node project files. It does not contain the reference solution — the appendix snippets on each lecture page are the only answer key.

First run:

unzip starter.zip
cd claude-code-nano
npm install
./ok 00

Expect ./ok 00 to fail on one test:

× Problem 0: warmup() must return 61001 (fix the off-by-two in index.ts)
AssertionError: expected 60999 to be 61001

That failure is the shipped, pre-course state — it’s Problem 0 in Lecture 01. The other two setup tests (Node version, importing index.ts doesn’t crash) already pass. Fixing warmup() so ./ok 00 goes green confirms your toolchain works end to end, and you’re off.

No API key is needed for any ./ok run — every test scripts a fake model, so the whole suite runs offline and free. You only need an ANTHROPIC_API_KEY once, for Lecture 06’s live acceptance run.
